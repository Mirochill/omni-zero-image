# Roadmap

## Phase 1: Software Prototype

- Implement package, CLI, architecture, tests, and remote CI.
- Keep draft mode available for instant integration tests.

## Phase 2: Tiny Remote Training

- Train on a small licensed dataset.
- Prove loss decreases.
- Generate first non-random text-to-image and image-to-image samples.

## Phase 3: Quality Model

- Train a larger teacher with curated broad-domain data.
- Add high-quality captions and edit instruction data.
- Evaluate prompt adherence, aesthetics, typography, and edit locality.

## Phase 4: Distillation

- Distill teacher into a 1-4 step student.
- Quantize and export.
- Benchmark cost and latency.

## Phase 5: Release

- Publish checkpoint, model card, examples, benchmark report, and limitations.
- Create a GitHub release only after the validation checklist is satisfied.
